# API

The OpenAPI 3.1 spec is auto-generated by FastAPI from route decorators and Pydantic models, then served at runtime at `/openapi.json`. A static copy committed at the repo root powers Backstage's API entity.

## Endpoints

### `GET /health`

Liveness probe. Returns the service version and a static `ok` status.

```json
{ "status": "ok", "version": "0.1.0" }
```

```bash
curl http://localhost:${{ values.listenPort }}/health
```

### `POST /echo`

Echoes the request body verbatim. Useful for verifying request routing + Pydantic deserialisation.

Request body:

```json
{ "message": "hi" }
```

Response:

```json
{ "message": "hi" }
```

```bash
curl -X POST -H 'Content-Type: application/json' \
  -d '{"message":"hi"}' \
  http://localhost:${{ values.listenPort }}/echo
```

## Interactive explorers

- **Swagger UI**: `http://localhost:${{ values.listenPort }}/docs`
- **ReDoc**: `http://localhost:${{ values.listenPort }}/redoc`

## Refreshing the static `openapi.json`

Commit a fresh copy whenever you change a route so Backstage's API tab reflects the new shape:

```bash
uv run uvicorn app.main:app --port ${{ values.listenPort }} &
sleep 2
curl -s http://localhost:${{ values.listenPort }}/openapi.json | jq > openapi.json
kill %1
```
